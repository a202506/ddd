# DingClock SPEC

## 1. 产品定位

`DingClock` 现在是一个钉钉手动辅助登录工具，不再承担自动打卡、定时任务、失败告警或后台保活职责。

应用当前只保留两类能力：

1. 本地安全保存钉钉密码。
2. 在用户手动点击按钮后，启动钉钉并辅助完成“已记住账号”的密码登录流程。

## 2. 保留功能

### 2.1 密码配置

用户可在配置页录入钉钉密码。

规则如下：

1. 密码仅保存在本机。
2. 明文密码先经 Android Keystore 加密，再写入加密的 SharedPreferences。
3. 若配置页留空，则沿用之前已保存的密码。
4. 首页可执行一次“验证密码可解密”，仅用于确认本机密钥和存储状态正常。

### 2.2 系统权限引导

配置页保留“系统权限”页面，用于引导用户检查下列设置：

1. 无障碍服务。
2. 忽略电池优化。
3. 厂商权限页，例如 vivo 的自启动、后台弹出界面等限制项。

其中无障碍服务为核心依赖，其余项目属于兼容性检查项。

### 2.3 手动启动钉钉并辅助登录

首页提供“启动钉钉并辅助登录”按钮，执行链路如下：

1. 检查无障碍服务是否已启用。
2. 对 `https://www.dingtalk.com/` 进行一次轻量网络探测。
3. 启动钉钉并等待其进入前台。
4. 识别当前页面：
   登录页。
   首页。
   考勤页。
   已打卡成功页。
   未知页。
5. 如果是登录页，则按既定无障碍流程依次点击“下一步”“同意并登录”“密码登录”，填入已保存密码并点击“登录”。
6. 如果不是登录页，则仅向用户展示识别结果，不再继续任何自动操作。

## 3. 已移除功能

以下能力已从当前实现中移除：

1. 自动打卡。
2. 上下班时间配置。
3. 节假日与随机抖动配置。
4. 自动排期、精确闹钟、WorkManager 兜底执行。
5. 前台打卡服务、开机重排、失败重试。
6. 打卡记录数据库。
7. 钉钉机器人 webhook 通知。

## 4. 持久化数据

当前仅持久化一个核心配置项：

1. 加密后的钉钉密码 `passwordCipher`。

## 5. 限制

1. 依赖钉钉客户端已记住登录账号，应用不会输入手机号。
2. 不处理短信验证码、滑块、人机校验等更强验证流程。
3. 不保证适配所有 ROM 和所有钉钉 UI 版本。
4. 应用不会自动打开考勤页，也不会执行任何打卡动作。
